## Design Philosophy

1. **Write less.** Every doc earns its place.
2. **One module, one doc.** If a doc covers two things, split it.
3. **Code is source of truth.** Docs explain *why*, code shows *what*.
4. **Refactor-friendly.** Small files, loose coupling, easy to rearrange.
